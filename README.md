# hw_relu_and_sigmoid
hardware implementation of Relu and sigmoid functions using a mux and piecewise linear approx's , respectively.
# Verilog Activation Functions: ReLU, Piecewise-Linear Sigmoid, and Tanh

## Abstract

This repository implements three hardware-friendly activation functions in Verilog:

1. `relu_mux`: a ReLU block constructed exactly as a 2x1 multiplexer with `in_data` and `0` as data inputs and the input MSB as the select line.
2. `sigmoid_pwl`: a fixed-point piecewise-linear approximation of the sigmoid function over the range `[-6, 6]`.
3. `tanh_from_sigmoid`: a tanh block derived from the implemented sigmoid using `tanh(x) = 2*sigma(2x) - 1`.

The sigmoid design targets a practical balance between approximation accuracy and hardware cost. A small design-space study was carried out by varying the number of linear pieces and measuring the resulting approximation error. Based on that study, the implementation in this repository uses five linear regions on the positive half-axis and mirrors them to the negative side.

## Repository Structure

- `rtl/relu_mux.v`: ReLU using a 2x1 mux formulation.
- `rtl/sigmoid_pwl.v`: piecewise-linear sigmoid in `Q4.12` fixed-point format.
- `rtl/tanh_from_sigmoid.v`: tanh implemented from the sigmoid module with shift/add logic.
- `tb/tb_relu_mux.v`: simple functional testbench for ReLU.
- `tb/tb_sigmoid_pwl.v`: functional spot-check testbench for sigmoid.
- `tb/tb_tanh_from_sigmoid.v`: functional spot-check testbench for tanh.

## Fixed-Point Format

Both modules use signed 16-bit data. The sigmoid module assumes `Q4.12` format:

- integer range: approximately `[-8, 7.9998]`
- fractional precision: `2^-12 = 0.000244140625`
- `1.0` is represented as `4096`
- `6.0` is represented as `24576`

This format was chosen because it comfortably covers the requested input range `[-6, 6]` while keeping coefficient storage and arithmetic simple.

## 1. ReLU Using a 2x1 Multiplexer

For a two's-complement signed input, the MSB is `1` for negative numbers and `0` for non-negative numbers. Therefore, the ReLU operation

`ReLU(x) = max(0, x)`

can be implemented as a 2x1 mux:

- input `0`: `in_data`
- input `1`: `0`
- select: `in_data[WIDTH-1]`

So the mux outputs:

- `0` when the input is negative
- the original input otherwise

This gives a very compact realization with no comparator tree required.

## 2. Piecewise-Linear Sigmoid Approximation

The true sigmoid is:

`sigma(x) = 1 / (1 + e^(-x))`

Direct exponential evaluation is expensive in hardware, so this project approximates the curve with line segments. The implementation uses the identity:

`sigma(-x) = 1 - sigma(x)`

Only the positive half (`[0, 6]`) is approximated directly. The negative half is generated by symmetry. Inputs outside the target range are saturated:

- `x <= -6` maps to `0`
- `x >= 6` maps to `1`

### Implemented Segmentation

The final implementation uses five positive-side segments:

| Region | Linear model |
| --- | --- |
| `0 <= x < 1` | `0.2327x + 0.5039` |
| `1 <= x < 2` | `0.1494x + 0.5894` |
| `2 <= x < 3` | `0.0710x + 0.7439` |
| `3 <= x < 4.5` | `0.0234x + 0.8867` |
| `4.5 <= x < 6` | `0.0054x + 0.9656` |

Quantized `Q4.12` coefficients used in hardware:

| Region | Slope (`m`) | Intercept (`c`) |
| --- | ---: | ---: |
| `[0, 1)` | `953` | `2064` |
| `[1, 2)` | `612` | `2414` |
| `[2, 3)` | `291` | `3047` |
| `[3, 4.5)` | `96` | `3632` |
| `[4.5, 6)` | `22` | `3955` |

The output is clamped to `[0, 1]` after evaluation.

## Experimental Study

I evaluated several symmetric piecewise-linear approximations over `[-6, 6]` before choosing the final implementation. Error was measured against the true floating-point sigmoid.

### Candidate Configurations

| Positive-side pieces | Total linear pieces with symmetry | Breakpoints on `[0, 6]` | Max abs. error | RMSE |
| ---: | ---: | --- | ---: | ---: |
| 1 | 2 | `6` | `0.1737` | `0.0639` |
| 2 | 4 | `3, 6` | `0.0589` | `0.0199` |
| 3 | 6 | `2, 4, 6` | `0.0290` | `0.0080` |
| 4 | 8 | `1.5, 3, 4.5, 6` | `0.0152` | `0.0044` |
| 5 | 10 | `1, 2, 3, 4.5, 6` | `0.0078` | `0.0021` |
| 6 | 12 | `0.75, 1.5, 2.5, 3.5, 4.5, 6` | `0.0069` | `0.0016` |

### Tradeoffs

- Moving from 1 to 3 positive-side segments gives the biggest accuracy improvement.
- Moving from 5 to 6 positive-side segments gives only a small extra reduction in error.
- More segments improve fidelity, but they also require more interval comparisons, more coefficient storage, and more mux/control logic.
- The 5-piece design is a strong middle point: it is much more accurate than 3 pieces, only marginally worse than 6 pieces, and still straightforward to implement and verify in Verilog.

For that reason, the repository uses the 5-piece version as the main implementation.

## Hardware Interpretation

The sigmoid datapath is simple:

1. Detect sign and compute absolute input magnitude.
2. Select one coefficient pair based on the input interval.
3. Compute `m*x + c` using fixed-point arithmetic.
4. Clamp to `[0, 1]`.
5. Mirror the result for negative inputs using `1 - y`.

This structure is synthesis-friendly and maps naturally to simple arithmetic hardware.

## 3. Tanh from Sigmoid

The hyperbolic tangent can be written as:

`tanh(x) = 2*sigma(2x) - 1`

This makes tanh especially convenient in fixed-point hardware when a sigmoid block already exists.

### Why No Extra Multiplier Is Needed

In this project, the `tanh_from_sigmoid` wrapper does not introduce any new multiplier hardware:

- multiplying the input by `2` is just a left shift
- multiplying the sigmoid output by `2` is also a left shift
- subtracting `1` is a subtraction by the fixed `Q4.12` constant `4096`

The only multipliers involved remain the ones already inside `sigmoid_pwl` for the piecewise-linear segment evaluation.

### Valid Input Range

Because the sigmoid core is designed for `[-6, 6]`, the tanh wrapper should be interpreted over `[-3, 3]` when using the identity above:

- `x <= -3` saturates close to `-1`
- `x >= 3` saturates close to `+1`
